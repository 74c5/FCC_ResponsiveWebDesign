Introduction

What follows is a lists of commands that I find most useful for everyday use on my linux desktop.

This is a hardly an exhaustive list of linux CLIs, but rather ones that I notice that I am using often.

Installing Updates

These are Ubuntu/Debian) specific instructions...

A list of updates can be fetched using:

$ sudo apt update

and installed with:

$ sudo apt upgrade

Redundant packages can be removed using:

$ sudo apt autoremove

I also have several flatpak applications installed, so updating also requires:

$ flatpak update

When I ran an arch based system and used Yay as my Aur helper, I could use just a single command:

$ yay -Syua

Of course, this might take just as long (or longer) since I was would have to re-compile most of my Aur applications.

Listing Files

I use the ls command with the switches als by default, e.g.:

$ ls -als

In fact it's muscle memory now. For 95% of my usecases this gives me all the information, I could ever want to know about a file.


The switches have the following effects:

  • -a lists all entries in the folder, even those beginning with . and ..
  • -l uses long list format. Each file gets its own line with columns for permissions, ownership and last date of modification.
  • -s prints the size of the file in blocks
Aliasing

I usually put aliases in or trigger alias creation from the last section of my .bashrc (or similar) file. These alias serve several purposes, namely:

  • Abbreviating longer commands making them faster to use and easier to remember.
  • Creating shortcut commands for changing to common directories.
  • Making commands from my own custom scripts (usually under development) available on the command line. (Yes, it's a bit of a hack, but I prefer it over updating my path.

Examples/Usecases:

  • I alias mounting of another windows computer at home to a short commmand, I only mount this file system to facilitate backups and so don't need it all the time. I can't use auto mount, since I have to ensure that the other computer is turned on before issuing the command.
  • I have shortcuts like cdprojx which will jump me to my project directory, which is often on a separate partition or drive.

E.g.

alias cdprojx="cd /media/data/projects/project X"

means that I only need to type projx to change to my "project X" working directory

Ping

Mosty, I ping a public DNS to test that my connection is up.

$ ping 1.1.1.1
